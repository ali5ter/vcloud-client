POST /admin/extension/blockingTask/{id}/action/updateProgress

Operation:
POST /admin/extension/blockingTask/{id}/action/updateProgress
Description:
This operation updates:
  • the message appended to task log
  • the timeout date - the new value is calculated as the current time plus the new timeout period passed to this method
The new timeout date value is calculated as the current time plus the new timeout period passed to this method. Operation succeeds (returns true) if request exists and is in status {@link TaskExtensionRequestStatusType#ACTIVE}. Upon successful completion this operation returns 204 No Content. Otherwise returns 409 Conflict if the status is not{@link TaskExtensionRequestStatusType#ACTIVE} or 404 Not Found if the supplied id for this task extension request is invalid.
Since:
1.5
Input parameters
{@link JAXBElement} represents message and new timeout date to this operation. Specify null for {@link TaskExtensionRequestUpdateProgressParamsType#TimeoutValueInMilliseconds} if there is no need to change the timeout of the task extension request. return Returns the updated task extension request.

Consume media type(s):
application/vnd.vmware.admin.blockingTaskUpdateProgressOperationParams+xml
Input type:
BlockingTaskUpdateProgressParamsType
Output parameters
Produce media type(s):
application/vnd.vmware.admin.blockingTask+xml
Output type:
BlockingTaskType
