POST /admin/extension/providervdcs

Operation:
POST /admin/extension/providervdcs
Description:
Creates a provider vDC. To be able to do this you should have a new resource pool from VC which is not a child or a sub child of an existing provider vDC. Also you should specify a data store and the VC server on which all these resources exists. The supplied information which you will need are the MoRefs of the resource pool and data store. The state of provider vDC resource which is returned is represented by its attribute status and will be initially 0 - not ready. The user of API should wait the task to be executed before continue to use the provider vDC. If there is an error while creating the vDC then the error description will be described in the . The resource pools and data stores which could be used could be found in this operation: GET .../api/v1.0/admin/extension/vimServer//resourcePoolList
Since:
0.9
